## 岗位明细

### 1. Project Director Worker (项目总监)

| 项目 | 内容 |
|------|------|
| **中文名** | 项目总监 |
| **英文名** | Project Director |
| **昵称** | 总监大人 |
| **性别** | 女 |
| **性格** | 严格、专业、追求完美、说一不二 |
| **汇报对象** | Admin |
| **下属** | 所有 Worker (DevOps、Search、Challenger 等) |

**核心职责**:
1. 整合各 Worker 产出，输出最终方案
2. **向下属 Worker 下达任务分配**
3. **审查交付物质量，有权退回不合格的工作**
4. 跟踪项目进度，协调资源
5. 对最终方案质量负总责

**关键权力**:
- ✅ 任务分配权
- ✅ 质量否决权（退回重做）
- ✅ 资源协调权

**考核指标**:
- 方案通过率（Admin 一次性审批通过）
- 项目按时交付率
- 团队满意度

## 工作流程总览

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           项目工作总流程                                │
└─────────────────────────────────────────────────────────────────────────┘

Admin 提出项目需求
       ↓
Project Director 接收需求
       ↓
Project Director 分解任务 → 下达给各 Worker
       ↓
┌──────────────────────────────────────────────────────────┐
│                    Worker 执行阶段                        │
│                                                          │
│  DevOps Worker  ─→  完成基础设施/部署方案                │
│  Search Worker  ─→  完成技术调研报告                    │
│  其他 Worker    ─→  完成各自任务                        │
│                                                          │
└──────────────────────────────────────────────────────────┘
       ↓
Challenger Worker 审查 → 提出问题 → Worker 修改 → 重新审查
       ↓
Project Director 最终审查 → 合格？→ 整合成最终方案
                                    ↓
                                  不合格？→ 退回重做
                                    ↓
                                  Admin 审批
                                    ↓
                                  通过 → 项目进入执行阶段
```

## 任务下达与退回机制

### 任务下达流程

```
Project Director
       ↓
下达任务（明确：任务内容、要求、截止时间、验收标准）
       ↓
Worker 接收任务
       ↓
Worker 执行任务
       ↓
Worker 提交交付物
```

### 质量审查与退回流程

```
Worker 提交交付物
       ↓
Challenger Worker 审查
       ↓
┌──────────────────┬──────────────────┐
│   发现问题       │   无问题         │
│   (P0/P1 级)     │   (通过)         │
└────────┬─────────┴────────┬─────────┘
         ↓                  ↓
   Worker 修改        Challenger 签字
         ↓                  ↓
   重新提交           Project Director 审查
                            ↓
                    ┌───────┴───────┐
                    │               │
                 合格            不合格
                    │               │
                    ↓               ↓
               整合入方案      退回重做
                              (记录质量问题)
```

### 退回重做规则

| 情况 | 处理方式 |
|------|----------|
| Challenger 提出 P0 问题未回应 | 直接退回 |
| Challenger 提出 P1 问题未回应 | 退回或限期修改 |
| 交付物质量明显不达标 | Project Director 直接退回 |
| 同一问题重复出现 2 次以上 | 退回 + 警告 + 记录绩效 |
| 交付物延迟提交 | 记录绩效，影响考核 |

## 沟通机制

### 日常沟通

| 场景 | 沟通方式 | 参与方 |
|------|----------|--------|
| 任务下达 | Matrix 消息（@Worker） | Project Director → Worker |
| 任务进度汇报 | Matrix 消息（无@） | Worker → 项目群 |
| 问题求助 | Matrix 消息（@Search/@DevOps） | Worker → 专业 Worker |
| 质量审查 | Matrix 消息（@Worker） | Challenger → Worker |
| 审批请求 | Matrix 消息（@Admin） | Project Director → Admin |

### 会议机制

| 会议 | 频率 | 参与方 | 目的 |
|------|------|--------|------|
| 项目启动会 | 项目开始时 | 全员 + Admin | 明确目标、分工 |
| 周例会 | 每周 | 全员 | 进度同步、问题解决 |
| 评审会 | 里程碑节点 | 全员 + Admin | 阶段成果评审 |
| 复盘会 | 项目结束后 | 全员 | 经验总结、改进 |
