## 2. 技术选型

| 模块 | 选型 | 理由 |
|------|------|------|
| 向量库 | ChromaDB | 轻量、快速集成、Python 友好 |
| 文件存储 | 本地文件系统 | MVP 阶段最简单 |
| 定时任务 | Celery + Redis | 成熟稳定 |
| 后端框架 | FastAPI | Python 异步、性能好 |
| 数据库 | PostgreSQL | 功能完整、支持 JSONB |

## 3. 数据流转流程

```
1. 用户提供链接
   ↓
2. 链接进入待处理队列 (PostgreSQL)
   ↓
3. 定时任务触发 (Celery)
   ↓
4. Crawler 抓取内容
   ├→ 文字内容 → Markdown
   ├→ 图片 → 本地存储
   └→ 代码 → 代码块
   ↓
5. Processor 处理内容
   ├→ 提取关键概念
   ├→ 生成摘要
   └→ 结构化存储
   ↓
6. ChromaDB 索引
   ├→ 向量化
   └→ 语义搜索
   ↓
7. 用户可访问
   ├→ 浏览模式
   └→ 问答模式
```

## 4. 交叉引用（超链接）实现

### 概念提取
- 使用 LLM 从内容中提取关键概念
- 存入 `concepts` 表

### 超链接生成
- Markdown 格式：`[[概念名]]`
- 前端解析为可点击链接

### 知识图谱
- `concept_relations` 表存储概念关系
- 支持查询相关概念
