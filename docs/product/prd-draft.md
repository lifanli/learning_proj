## 1. 产品定位

**智能出版学习系统** — 智能体既是出版社（编写高质量教材）又是作者（答疑互动）

### 核心价值
- 降低跨平台学习成本（GitHub/微信/arXiv → 统一知识库）
- 渐进式学习路径（新手友好）
- 双向交互（浏览书籍 + 自然对话）

---

## 2. 目标用户

- 深度学习/机器学习/数学/物理 学习者
- 需要跨平台整理学习资料的研究人员
- 希望有系统性学习路径的自学者

---

## 3. MVP 功能范围

### Must Have（第一版必须做）
- [ ] 内容采集（GitHub/arXiv/微信公众号）
- [ ] 内容处理（文字提取/图片存储/摘要生成）
- [ ] 知识库存储（Markdown + 元数据 + 向量索引）
- [ ] 搜索功能（关键词 + 向量）
- [ ] Matrix 机器人交互

### Nice to Have（第二版再做）
- [ ] 学习路径推荐
- [ ] 用户进度追踪
- [ ] 知乎/CSDN/博客园抓取
- [ ] PDF 全文解析
- [ ] 浏览器插件
- [ ] 导出功能

---

## 4. 用户完整流程

### 4.1 添加内容源
```
用户 → 提交链接 → 系统抓取 → 内容处理 → 存入知识库
```

### 4.2 浏览知识库
```
用户 → 浏览目录 → 查看章节 → 点击超链接 → 相关知识
```

### 4.3 搜索查询
```
用户 → 提问 → 向量搜索 → 返回相关内容 + 来源
```

---

## 5. 微信公众号方案

**推荐方案**：用户复制链接 → 通用爬虫抓取

**理由**：
- 快速启动，无需额外开发
- 法律风险低

**后续优化**：开发浏览器插件（一键保存）

---

## 6. 渐进式学习路径

### 冷启动方案
- 新用户自评水平（新手/进阶/高级）
- 根据自评推荐对应难度内容

### 学习路径生成
- 基于概念难度和前置关系
- 动态调整（根据用户学习进度）

---

## 7. 验收标准

### 功能验收
- [ ] 可以添加 GitHub/arXiv/微信链接
- [ ] 内容处理后存入知识库
- [ ] 支持关键词和向量搜索
- [ ] Matrix 机器人可交互

### 性能验收
- [ ] 单个链接处理时间 < 5 分钟
- [ ] 搜索响应时间 < 1 秒
