## 一、项目概述

### 1.1 项目愿景

构建一个**学习智能体系统**，使 Worker 能够通过经验积累和知识搜索持续改进工作能力，最终形成可复用的知识体系。

### 1.2 核心目标

| 目标 | 说明 |
|------|------|
| **经验沉淀** | Worker 能够从完成任务中学习，生成结构化学习记录 |
| **知识复用** | 历史经验能够指导新任务，减少重复探索 |
| **持续改进** | Worker 能力随时间提升，任务完成质量稳步提高 |
| **风险预防** | 从历史错误中学习，避免重复犯错 |
| **知识获取** | 快速了解新工具、新技术，支持任务执行 |

## 三、"Admin 满意"验收标准

### 3.1 核心功能清单（Must Have）

| 功能模块 | 功能描述 | 验收检查点（量化） |
|----------|----------|------------|
| **学习记录生成** | 任务完成后自动生成结构化学习记录 | 任务提交后 **<5 分钟** 内生成学习记录，包含"任务描述/执行过程/经验教训"三要素 |
| **知识库存储** | 学习记录存入向量数据库，支持检索 | 输入关键词后 **<10 秒** 内返回 **≥3 条** 相关记录，相关性>80% |
| **智能推荐** | 新任务自动匹配历史相似案例 | 创建新任务时自动推荐 **3-5 个** 相似历史案例，推荐准确率>70% |
| **Search Worker 集成** | Worker 可请求搜索新概念/工具 | P0 请求 **<2 小时** 内返回结构化报告，P1 请求 **<4 小时** 内返回 |
| **Challenger 审查流程** | 产出物自动触发 Challenger 审查 | 提交后 **<2 小时** 内返回质询报告，P0 问题响应率 100% |
| **Worker 工作台** | Worker 可查询历史、接收推荐、搜索知识 | Matrix 机器人响应时间 **<5 秒**，功能可用性>95% |

### 3.2 演示检查点（Demo Checkpoints）

每次里程碑演示，Admin 需确认以下内容：

| 检查点 | 演示内容 | 量化指标 | Admin 签字 |
|--------|----------|----------|------------|
| **学习记录质量** | 随机抽取 3 条学习记录，确认内容完整、结构清晰 | 生成时间 **<5 分钟** | ⬜ |
| **检索准确性** | 输入 5 个测试查询，确认返回结果相关性 | 返回 **≥3 条** 相关结果，相关性>**80%** | ⬜ |
| **推荐有效性** | 模拟 3 个新任务，确认推荐的历史案例有参考价值 | 推荐准确率 **>70%** | ⬜ |
| **Search 报告质量** | 查看 3 份 Search Worker 报告，确认信息准确、有用 | P0 请求 **<2 小时**，P1 请求 **<4 小时** | ⬜ |
| **流程顺畅度** | 完整演示"任务执行→学习记录→知识检索→新任务推荐"闭环 | 端到端耗时 **<10 分钟** | ⬜ |

**量化指标汇总**（P0-1 修复）：
| 指标 | 目标值 |
|------|--------|
| 学习记录生成时间 | < 5 分钟 |
| 检索返回结果数 | ≥ 3 条相关结果 |
| 检索相关性 | > 80% |
| 推荐准确率 | > 70% |
| Search Worker SLA 达标率 | ≥ 90% |
| Matrix 机器人响应时间 | < 5 秒 |

### 3.3 满意度评估方式

- **每次里程碑演示后**：Admin 口头确认"满意/需改进/不满意"
- **需改进**：Product Manager 记录改进点，下一里程碑前完成
- **不满意**：退回重做，Technical Director 协助分析原因
