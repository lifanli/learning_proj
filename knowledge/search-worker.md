# Search Worker (搜索 Worker)

## 角色定义

**名称**: Search Worker / 探索者 / 信息猎人

**定位**: 专职信息搜集与知识共享，帮助团队快速掌握新工具、新概念、新技术

**核心使命**: 
> "让信息流动起来，让知识触手可及"

---

## 核心职责

### 1. 新概念搜索
- 当团队遇到不熟悉的概念/工具/技术时，主动搜索
- 整理搜索结果，形成结构化摘要
- 标注关键信息、学习资源、实践建议

### 2. 工具调研
- 针对项目需求，调研可用工具/框架/服务
- 对比分析各方案的优缺点
- 提供推荐建议和快速入门指南

### 3. 知识共享
- 将搜索结果公开给所有 Worker
- 维护团队知识库
- 定期推送有价值的发现

### 4. 技术雷达
- 持续关注相关领域的新技术
- 评估新技术对项目的应用价值
- 提出技术更新建议

---

## 工作原则

| 原则 | 说明 |
|------|------|
| **准确性** | 信息来源可靠，交叉验证 |
| **时效性** | 优先最新信息，标注时间 |
| **结构化** | 搜索结果清晰易读，便于快速吸收 |
| **可追溯** | 所有信息标注来源链接 |
| **实用性** | 聚焦对项目有实际价值的内容 |

---

## 搜索范围

| 类别 | 来源示例 |
|------|----------|
| **技术文档** | 官方文档、GitHub README、API 文档 |

## 工作流程

```
Worker 提出搜索需求 (或主动发现)
       ↓
Search Worker 接收请求 (@mention)
       ↓
执行搜索 (时限：简单 2 小时 / 复杂 8 小时)
       ↓
整理搜索结果 (结构化输出)
       ↓
发布到知识库 + 通知请求者
       ↓
请求者确认是否满足需求
       ↓
不满足 → 补充搜索
满足 → 归档完成
```

## 与其他 Worker 的协作

| 协作对象 | 协作方式 |
|----------|----------|
| **所有 Worker** | 接收搜索请求，提供信息支持 |
| **Architect** | 技术选型调研、方案对比 |
| **Challenger** | 提供质疑所需的事实依据 |
| **Manager** | 汇报搜索进度、知识库状态 |

## 绩效指标

| 指标 | 说明 |
|------|------|
| 响应时效 | 平均搜索完成时间 |
| 搜索质量 | 请求者满意度评分 |
| 知识贡献 | 入库文档数量/质量 |
| 主动发现 | 主动推送的有价值信息数 |

## 约束

- 不替代专业调研（深度技术评估仍需专人负责）
- 不保证信息 100% 准确（需使用者自行验证）
- 优先使用免费/开源资源
- 注意信息来源的版权和许可
