## 任务下达与审核流程

```
┌─────────────────────────────────────────────────────────────┐
│                    任务下达与审核流程                        │
└─────────────────────────────────────────────────────────────┘

Project Director 下达任务
       ↓
明确：任务内容、截止时间、交付标准
       ↓
Worker 接收任务并开始执行
       ↓
Worker 提交产出
       ↓
Project Director 审核
       ↓
    ┌──────────────┐
    │   是否合格？  │
    └──────┬───────┘
           │
     ┌─────┴─────┐
     │           │
    是          否
     │           │
     ↓           ↓
  通过      退回重做
     │           │
     │           ↓
     │      Worker 修改
     │           │
     │           ↓
     └──────←──┘
           (重新提交)
     │
     ↓
整合入最终方案
```

## 命名约定

### 中文名命名规则
- **简洁易记**：2-4 个字为宜
- **体现职责**：名字能反映岗位特点
- **风格统一**：整体风格协调

### 建议命名方向
| 角色 | 命名方向 | 示例 |
|------|----------|------|
| Architect | 智慧、规划类 | 智谋、架构、规划师 |
| Search Worker | 探索、信息类 | 探索者、知觅、信息官 |
| Challenger | 质疑、审查类 | 质询者、审思、挑刺官 |
| Project Director | 管理、统筹类 | 总监、统筹、掌舵人 |
| Frontend Worker | 视觉、界面类 | 视觉师、界面师 |
| Backend Worker | 核心、引擎类 | 引擎师、核心师 |

**最终命名权归 Project Director 所有**
