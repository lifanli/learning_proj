## 📋 详细岗位职责

### 1️⃣ 老板 💕 (Admin)
```
职责：
- 项目发起人
- 最终决策者
- 审批最终方案

权力：
✅ 项目方向决策
✅ 最终方案审批
✅ 资源调配
```

### 2️⃣ 经理 💕 (Manager)
```
职责：
- 协调所有 Worker
- 向老板汇报进度
- 组织讨论

权力：
✅ Worker 调度
✅ 进度跟踪
✅ 信息汇总
```

### 3️⃣ 林总监 (Project Director) 👩‍💼
```
职责：
- 整合所有 Worker 的方案
- 向下属下达任务并设定截止时间
- 审核交付物质量（不合格直接退回）
- 输出最终落地方案给老板

权力：
✅ 任务分配权
✅ 截止时间设定权
✅ 质量审核权（可退回重做）
✅ 最终方案决策权

输出物：
- 最终落地方案文档
- 任务分配表
- 质量审核报告
```

### 4️⃣ 搜索小助手 (Search Specialist) 👧
```
职责：
- 调研技术选型（向量库/存储/工具等）
- 搜索新论文、新工具、新技术
- 整理对比报告（表格 + 推荐方案）
- 发布到共享知识库供大家学习

输出物：
- 技术对比报告
- 每日新发现汇总
- 专题调研报告
```

### 5️⃣ 反问大哥 (Critical Reviewer) 👨‍🦱
```
职责：
- 审查所有 Worker 提交的方案
- 从 10 个维度提出质疑（技术/安全/性能/成本等）
- 跟踪问题直到得到满意回答
- 输出风险评估报告

输出物：
- 方案审查报告
- 风险清单（高/中/低）
- 问题跟踪表
```

### 6️⃣ 架构师老张 (System Architect) 👨‍💻
```
职责：
- 设计整体技术架构
- 技术选型建议
- 数据流转流程设计
- 交叉引用（超链接）方案
- 回答反问大哥的技术质疑

输出物：
- 技术架构图
- 技术选型报告
- 数据流转流程图
```

### 7️⃣ 产品小姐姐 (Product Manager) 👩‍💼
```
职责：
- 定义产品定位和核心价值
- 设计用户完整使用流程
- 确定 MVP 功能范围
- 微信公众号处理方案
- 渐进式学习路径设计

输出物：
- 产品需求文档（PRD）
- 用户流程图
- MVP 功能清单
- 微信处理方案
```

### 8️⃣ 后端工程师 (Backend Engineer)
```
职责：
- API 设计（RESTful 端点）
- 数据库表结构设计
- 业务逻辑实现
- 定时任务设计
- 安全验证方案

输出物：
- API 设计文档
- 数据库 ER 图
- 代码结构说明
- 安全验证方案
```

### 9️⃣ 前端工程师 (Frontend Engineer)
```
职责：
- 界面设计（知识库浏览器）
- 交互设计（超链接/搜索）
- 技术选型（React/Vue 等）
- 学习进度展示方式

输出物：
- 界面设计稿
- 交互流程图
- 技术选型报告
```

### 🔟 代码审查员 (Code Reviewer)
```
职责：
- 制定代码规范
- 安全审查（爬虫/文件上传等）
- 测试策略设计
- 性能优化建议

输出物：
- 代码规范文档
- 安全清单
- 测试策略
```

### 1️⃣1️⃣ 运维工程师 (DevOps Engineer)
```
职责：
- 部署架构设计
- 容器化方案（Docker）
- 监控和日志
- 备份策略

输出物：
- 部署架构图
- Docker 配置
- 监控方案
```
