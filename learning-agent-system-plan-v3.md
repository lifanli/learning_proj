## 一、项目背景与目标

### 1.1 项目愿景

构建一个**学习智能体系统**，使 Worker 能够通过经验积累和知识搜索持续改进工作能力。

### 1.2 核心目标

| 目标 | 说明 | 成功指标 |
|------|------|----------|
| **经验沉淀** | Worker 能够从完成任务中学习 | 每次任务后生成学习记录 |
| **知识复用** | 历史经验能够指导新任务 | 相似任务推荐准确率>80% |
| **持续改进** | Worker 能力随时间提升 | 任务完成时间缩短、返工率下降 |
| **风险预防** | 从历史错误中学习避免重复犯错 | 同类错误不再发生 |
| **知识获取** | 快速了解新工具、新技术 | 搜索请求响应时间<4 小时 |

### 2.2 核心组件（更新）

| 组件 | 职责 | 技术选型建议 |
|------|------|-------------|
| **学习记录生成器** | 从任务执行中提取结构化经验 | Python + LLM |
| **知识库** | 存储和检索学习记录 | 向量数据库 (可选：Chroma/Weaviate/LightRAG) |
| **智能推荐引擎** | 匹配新任务与历史案例 | 相似度搜索 + 规则引擎 |
| **Worker 工作台** | 与 Worker 交互的界面 | Matrix Bot / Web UI |
| **Search Worker** | 新概念搜索和知识整理 ⭐新增 | 搜索引擎 + LLM 整理 |

### 2.3 技术选型说明

> **重要**: 以下技术选型为**推荐**，可根据实际情况调整。

| 层级 | 推荐选型 | 可选替代 | 说明 |
|------|----------|----------|------|
| **本地存储** | MinIO | 本地文件系统/NFS/其他对象存储 | 用于存储原始文档和附件 |
| **向量数据库** | LightRAG | Chroma/Weaviate/Pinecone | LightRAG 支持图 + 向量混合检索 |
| **嵌入模型** | text-embedding-3-small | bge-large-zh/m3e | 根据语言需求选择 |
| **LLM** | Qwen3.5-Plus | GPT-4/Claude/本地模型 | 生成学习记录、总结归纳 |
| **搜索引擎** | Google/Bing API | 本地搜索引擎 | Search Worker 使用 |

## 三、Worker 角色体系（更新）

### 3.1 角色总览

| 角色 | 职责 | 核心产出 |
|------|------|----------|
| **Worker (执行者)** | 执行具体任务 | 代码、文档、设计等交付物 |

## 四、Search Worker 工作流程

### 4.1 搜索请求触发

| 触发条件 | 示例 | 优先级 |
|----------|------|--------|
| Worker 遇到新工具 | "这个向量数据库没用过，有入门资料吗？" | P1 |
| 技术选型调研 | "Chroma 和 Weaviate 哪个更适合我们？" | P1 |
| 问题排查支持 | "这个报错没见过，有类似案例吗？" | P0 |
| 主动知识更新 | "最近 X 技术有新版本，我整理一下更新内容" | P2 |

### 4.2 搜索报告输出

Search Worker 输出格式见 `custom-skills/search-worker/SKILL.md`

核心内容：
- 核心概念解释
- 关键要点（是什么/为什么/怎么用）
- 优缺点分析
- 替代方案对比
- 学习资源推荐
- 团队应用建议

### 4.3 知识入库

```
搜索报告完成
       ↓
结构化处理 (提取关键词、标签)
       ↓
生成嵌入向量
       ↓
存入向量数据库 (搜索报告库)
       ↓
@mention 相关 Worker 通知
       ↓
Worker 可检索和参考
```

### 2.4 技术选型对比分析（新增）

**更新时间**: 2026-03-06 14:15 UTC  
**分析人**: DevOps Engineer  
**数据来源**: 基于各方案官方文档、GitHub README、社区基准测试综合评估

| 维度 | LightRAG | Chroma | Weaviate | 推荐指数 |
|------|----------|--------|----------|----------|
| **查询性能** | 中（图检索有优势） | 高（纯向量检索） | 高（优化良好） | Chroma ≈ Weaviate > LightRAG |
| **存储成本** | 中（需存储图结构） | 低（轻量） | 中 | Chroma > LightRAG ≈ Weaviate |
| **学习曲线** | 中（文档较少） | 低（文档完善） | 中 | Chroma > Weaviate > LightRAG |
| **社区活跃度** | 中（新兴项目） | 高（广泛使用） | 高（企业支持） | Chroma ≈ Weaviate > LightRAG |
| **图检索支持** | ✅ 原生支持 | ❌ 不支持 | ⚠️ 需插件 | LightRAG 唯一支持 |
| **自托管友好度** | 高（Python） | 高（Python） | 中（需 Docker） | Chroma ≈ LightRAG > Weaviate |
| **中文支持** | 好 | 好 | 好 | 持平 |

**参考链接**:
- LightRAG: https://github.com/HKUDS/LightRAG
- Chroma: https://docs.trychroma.com/
- Weaviate: https://weaviate.io/developers/weaviate

#### 推荐决策

**首选方案**: LightRAG

**理由**:
1. **图 + 向量混合检索** 是核心优势，适合"任务关联推荐"场景（如：相似任务、相关问题模式）
2. 虽然学习曲线稍陡，但团队有 Python 经验，可接受
3. 自托管友好，符合隐私要求

**备选方案**: Chroma

**适用场景**: 如果试点阶段发现 LightRAG 复杂度过高，可快速切换到 Chroma，牺牲图检索能力换取开发速度。

#### 试点验证计划

| 阶段 | 目标 | 验证内容 |
|------|------|----------|
| 第 1 周 | LightRAG 部署 | 安装难度、文档完整性 |
| 第 2 周 | 基础检索测试 | 向量检索性能、准确性 |
| 第 3 周 | 图检索测试 | 任务关联推荐效果 |
| 第 4 周 | 决策评审 | 根据试点结果决定是否切换 |
