## 一、项目背景与目标

### 1.1 项目愿景

构建一个**学习智能体系统**，使 Worker 能够通过经验积累持续改进工作能力。

### 1.2 核心目标

| 目标 | 说明 | 成功指标 |
|------|------|----------|
| **经验沉淀** | Worker 能够从完成任务中学习 | 每次任务后生成学习记录 |
| **知识复用** | 历史经验能够指导新任务 | 相似任务推荐准确率>80% |
| **持续改进** | Worker 能力随时间提升 | 任务完成时间缩短、返工率下降 |
| **风险预防** | 从历史错误中学习避免重复犯错 | 同类错误不再发生 |

```yaml
        理由: 为什么选 A
    遇到的问题:
      - 问题: 描述
        解决方案: 如何解决的

    是否达成目标: 是/部分/否

    时间评估: 提前/按时/延期

    成功经验:

    - 技术领域标签
    - 问题类型标签
    - 解决方案标签
    
  关联内容:
    相关任务: [task-xxx, task-yyy]
    参考文档: [链接]
    代码提交: [commit hash]
```

## 四、工作流程设计

### 4.1 学习闭环

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           学习智能体工作闭环                            │
└─────────────────────────────────────────────────────────────────────────┘

  ┌─────────────┐
  │  新任务到来  │
  └─────────────┘
       ↓
  ┌─────────────────────────────────┐
  │ 智能推荐引擎检索历史相似任务     │
  │ - 相似的技术栈                  │
  │ - 相似的问题类型                │
  │ - 成功的解决方案                │
  │ - 需要避免的错误                │
  └─────────────────────────────────┘
       ↓
  ┌─────────────┐
  │  Worker 参考 │
  │  历史经验   │
  └─────────────┘
       ↓
  ┌─────────────┐
  │  执行任务   │
  │  记录过程   │
  └─────────────┘
       ↓
  ┌─────────────┐
  │  Challenger │
  │  审查反馈   │
  └─────────────┘
       ↓
  ┌─────────────────────────────────┐
  │ 生成学习记录                    │
  │ - 提取关键决策和理由            │
  │ - 记录遇到的问题和解决方案      │
  │ - 总结成功经验和教训反思        │
  └─────────────────────────────────┘
       ↓
  ┌─────────────┐
  │ 存入知识库  │ → (循环回到开始)
  └─────────────┘
```

### 4.2 触发时机

| 时机 | 触发条件 | 执行动作 |
|------|----------|----------|
| **任务开始前** | Worker 接收新任务 | 推荐相似历史任务和经验 |
| **任务进行中** | Worker 遇到问题 | 推荐类似问题的解决方案 |
| **任务完成后** | Worker 提交成果 | 自动生成学习记录 |
| **Challenger 审查后** | 审查报告生成 | 更新学习记录中的反馈部分 |
| **定期复盘** | 每周/每月 | 分析学习记录，发现模式 |

## 五、技术实现方案

### 5.1 技术栈建议

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| **向量数据库** | Chroma / Weaviate | 轻量、易部署、支持本地运行 |
| **嵌入模型** | text-embedding-3-small | 成本低、效果好 |
| **LLM** | Qwen3.5-Plus / GPT-4 | 生成学习记录、总结归纳 |
| **后端服务** | Python FastAPI | 快速开发、生态丰富 |
| **前端界面** | React + TypeScript | 与前端技术栈一致 |
| **消息集成** | Matrix Bot | 与现有 Worker 通信渠道一致 |

### 5.2 部署架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        学习智能体系统部署架构                            │
└─────────────────────────────────────────────────────────────────────────┘

                    ┌─────────────────┐
                    │   Matrix 消息   │
                    │   (Worker 交互) │
                    └────────┬────────┘
                             │
                             ↓
                    ┌─────────────────┐
                    │   API Gateway   │
                    │   (FastAPI)     │
                    └────────┬────────┘
                             │
            ┌────────────────┼────────────────┐
            ↓                ↓                ↓
    ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
    │  学习记录    │ │  智能推荐    │ │  知识库      │
    │  生成服务    │ │  引擎服务    │ │  管理服务    │
    └──────┬───────┘ └──────┬───────┘ └──────┬───────┘
           │                │                │
           └────────────────┼────────────────┘
                            │
                            ↓
                   ┌─────────────────┐
                   │  向量数据库     │
                   │  (Chroma)       │
                   └─────────────────┘
```

### 5.3 数据流

```
Worker 任务完成
       ↓
触发学习记录生成
       ↓
LLM 分析任务内容 → 提取关键信息 → 生成结构化记录
       ↓
存入向量数据库 (带嵌入向量)
       ↓
新任务到来时检索相似记录
       ↓
返回 Top-N 相关经验给 Worker
```
