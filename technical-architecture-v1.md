## 一、项目概述

### 1.1 项目背景

构建一个智能学习助手 Web 应用，为用户提供知识梳理和问题解答服务。该系统基于 Vue3 前端、FastAPI 后端和 LangChain 智能体框架，提供智能化的学习体验。

### 1.2 核心目标

| 目标 | 说明 | 验收标准 |
|------|------|----------|
| **知识梳理** | 帮助用户整理和结构化知识 | 支持多种知识输入格式，自动生成知识图谱 |
| **问题解答** | 智能回答用户问题 | 回答准确率>85%，响应时间<3 秒 |
| **持续学习** | 系统能够从交互中学习 | 支持知识库更新和优化 |
| **用户友好** | 简洁易用的界面 | 用户满意度>4/5 |

### 1.3 技术栈确认

| 层级 | 技术选型 | 版本 | 说明 |
|------|----------|------|------|
| **前端** | Vue3 | 3.4+ | 响应式 UI 框架 |
| **前端状态管理** | Pinia | 2.1+ | Vue3 官方推荐状态管理 |
| **前端 UI 组件** | Element Plus | 2.5+ | Vue3 组件库 |
| **后端** | FastAPI | 0.109+ | 高性能 Python Web 框架 |
| **AI 框架** | LangChain | 0.1+ | 智能体开发框架 |
| **向量数据库** | LightRAG | latest | 图 + 向量混合检索 |
| **嵌入模型** | text-embedding-3-small | - | OpenAI 嵌入模型 |
| **LLM** | Qwen3.5-Plus | - | 通过 Higress Gateway 访问 |
| **部署** | Docker + WSL2 | - | 本地部署 |

## 三、技术选型分析

### 3.1 前端技术选型

#### Vue3 vs React vs Angular

| 维度 | Vue3 | React | Angular |
|------|------|-------|---------|
| **学习曲线** | 低 | 中 | 高 |
| **开发效率** | 高 | 高 | 中 |
| **生态成熟度** | 高 | 非常高 | 高 |
| **性能** | 优秀 | 优秀 | 良好 |
| **TypeScript 支持** | 优秀 | 优秀 | 优秀 |
| **团队熟悉度** | 高 | 中 | 低 |

**决策**: 选择 Vue3
- 学习曲线低，开发效率高
- 组合式 API 提供更好的代码组织
- Element Plus 提供丰富的组件库
- 符合"越快越好"的时间要求

#### 状态管理：Pinia vs Vuex

| 维度 | Pinia | Vuex |
|------|-------|------|
| **TypeScript 支持** | 原生优秀 | 需要配置 |
| **API 简洁度** | 简洁 | 较复杂 |
| **体积** | 1KB | 较大 |
| **Vue3 兼容性** | 官方推荐 | 维护模式 |

**决策**: 选择 Pinia
- Vue3 官方推荐
- 更好的 TypeScript 支持
- 更简洁的 API

### 3.2 后端技术选型

#### FastAPI vs Django vs Flask

| 维度 | FastAPI | Django | Flask |
|------|---------|--------|-------|
| **性能** | 非常高 | 中 | 中 |
| **异步支持** | 原生 | 有限 | 需要扩展 |
| **自动文档** | 内置 | 需要插件 | 需要插件 |
| **学习曲线** | 低 | 中 | 低 |
| **生态** | 快速增长 | 成熟 | 成熟 |
| **AI/ML 集成** | 优秀 | 良好 | 良好 |

**决策**: 选择 FastAPI
- 原生异步支持，性能优秀
- 自动生成 OpenAPI 文档
- 与 LangChain 集成良好
- 类型提示完善，开发体验好

### 3.3 AI 框架选型

#### LangChain vs LlamaIndex vs Haystack

| 维度 | LangChain | LlamaIndex | Haystack |
|------|-----------|------------|----------|
| **功能完整性** | 非常高 | 高 | 高 |
| **文档质量** | 优秀 | 良好 | 良好 |
| **社区活跃度** | 非常高 | 高 | 中 |
| **学习曲线** | 中 | 中 | 中高 |
| **RAG 支持** | 优秀 | 优秀 | 优秀 |
| **Agent 支持** | 优秀 | 有限 | 有限 |

**决策**: 选择 LangChain
- 功能最完整，支持 Agent、Chain、Tool
- 社区最活跃，文档完善
- 与多种向量数据库和 LLM 集成良好

### 3.4 向量数据库选型

见项目文档 `learning-agent-system-plan-v3.md` 第 2.4 节技术选型对比分析。

**决策**: 选择 LightRAG
- 图 + 向量混合检索是核心优势
- 适合知识关联推荐场景
- 自托管友好

### 4.3 数据模型设计

#### 4.3.1 核心实体

```python
# 用户模型
class User:
    id: UUID
    username: str
    email: str
    password_hash: str
    created_at: datetime
    updated_at: datetime

# 知识模型
class Knowledge:
    id: UUID
    title: str
    content: str
```
