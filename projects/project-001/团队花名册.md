## 团队协作流程

### 任务下达流程

```
项目总监 → 下达任务 → Worker → 确认接收 → 执行任务 → 提交交付物
                                              ↓
项目总监 ← 审核结果 ← 质量审查专员 ← 审查交付物
    ↓
通过 → 任务完成
    ↓
不通过 → 退回重做 → Worker 修改 → 重新提交
```

### 沟通规范

| 场景 | 沟通方式 | 说明 |
|------|----------|------|
| 任务下达 | Matrix @mention | 项目总监下达任务时使用 |
| 任务确认 | Matrix 回复 | Worker 确认接收任务 |
| 进度汇报 | Matrix 群内 | 定期汇报或里程碑完成时 |
| 问题求助 | Matrix @mention | 遇到困难时向上级求助 |
| 方案审查 | Matrix + 文档 | 审查报告 + 群内讨论 |
| 决策审批 | Matrix @admin | 需要管理员决策时 |

### 文档规范

| 文档类型 | 位置 | 命名规范 |
|----------|------|----------|
| 技术方案 | `shared/projects/{project-id}/` | `{主题}-plan.md` |
| 调研报告 | `shared/knowledge/search-results/` | `{主题}-intro.md` |
| 审查报告 | `shared/projects/{project-id}/` | `{主题}-review.md` |
| 任务文档 | `shared/tasks/{task-id}/` | `plan.md`, `result.md` |
| 进度报告 | `shared/projects/{project-id}/progress-reports/` | `{日期}.md` |

## 团队文化

### 核心价值观
1. **质量第一**：不接受、不制造、不传递缺陷
2. **开放沟通**：有话直说，对事不对人
3. **持续学习**：项目是学习的过程，共同成长
4. **相互尊重**：尊重每个人的专业和努力
5. **结果导向**：以交付高质量成果为目标

### 工作氛围
- **严格但友好**：工作标准严格，但团队氛围友好
- **批判但建设**：可以质疑，但要提出建设性意见
- **紧张但有序**：进度紧张，但工作有条不紊
- **女性领导力**：细腻、有同理心、坚定但不强硬
