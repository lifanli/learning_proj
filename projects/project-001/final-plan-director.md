## 执行摘要

本项目旨在构建一个**学习智能体系统**，通过引入三个专业 Worker 角色（Challenger 反问者、Search 搜索者、学习智能体），实现团队能力的持续改进和知识沉淀。

**核心决策**：
- 采用分阶段实施策略，先试点 Challenger Worker，再逐步引入其他角色
- 技术选型保持开放，不强制绑定特定方案（向量库可选 LightRAG/Chroma/Weaviate）
- 知识库访问权限采用分级制度，失败记录匿名化处理

**预期成果**：任务完成时间缩短 20%、返工率下降 30%、知识复用率>80%

---

## 一、项目背景与目标

### 1.1 背景

当前团队协作中存在以下痛点：
- 经验无法有效沉淀，同类错误重复发生
- 新工具/新技术学习成本高，信息获取效率低
- 方案质量参差不齐，缺乏系统性审查机制

### 1.2 核心目标

| 目标 | 说明 | 成功指标 (KPI) |
|------|------|----------------|
| **经验沉淀** | Worker 能够从完成任务中学习 | 每次任务后生成学习记录 (100% 覆盖) |
| **知识复用** | 历史经验能够指导新任务 | 相似任务推荐准确率>80% |
| **持续改进** | Worker 能力随时间提升 | 任务完成时间缩短 20%、返工率下降 30% |
| **风险预防** | 从历史错误中学习避免重复犯错 | 同类错误不再发生 |
| **知识获取** | 快速了解新工具、新技术 | 搜索请求响应时间<4 小时 |

### 1.3 范围

- **包含**:
  - Challenger Worker 角色设计与实施
  - Search Worker 角色设计与实施
  - 学习记录生成器开发
  - 知识库（向量数据库）部署
  - 智能推荐引擎开发
  - Worker 工作台开发

- **不包含**:
  - 业务代码开发
  - 现有 Worker 系统的重构
  - 第三方系统的大规模集成

### 2.2 技术选型

| 组件 | 推荐选型 | 备选方案 | 选择理由 |
|------|----------|----------|----------|
| **本地存储** | MinIO | 本地文件系统/NFS | 对象存储，与 S3 兼容，易扩展 |
| **向量数据库** | LightRAG | Chroma/Weaviate | 支持图 + 向量混合检索 (Admin 推荐，可调整) |
| **嵌入模型** | text-embedding-3-small | bge-large-zh/m3e | 成本低，效果好 |
| **LLM** | Qwen3.5-Plus | GPT-4/Claude | 当前网关配置，可调整 |
| **搜索引擎** | Google/Bing API | 本地搜索引擎 | Search Worker 使用 |
| **后端框架** | Python FastAPI | Flask | 快速开发，生态丰富 |
| **前端框架** | React + TypeScript | Vue | 与团队技术栈一致 |
