## 一、执行摘要

经过整合各 Worker 的方案和 Challenger 的质询，本项目包含**三个核心组成部分**：

| 组成部分 | 状态 | 说明 |
|----------|------|------|
| **1. 反问 Worker 机制** | ✅ 已完成设计 | 独立质量保障角色，审查所有 Worker 输出 |
| **2. 学习智能体系统** | ⏳ 待开发 | 经验沉淀和知识推荐系统 |
| **3. Search Worker** | ⏳ 待开发 | 新概念搜索和知识整理 |

**当前优先级**: 先启动反问 Worker 机制试点，同时确认学习智能体系统需求。

### 2.2 技术栈决策

| 层级 | 最终选型 | 决策理由 |
|------|----------|----------|
| **向量数据库** | LightRAG | 支持图 + 向量混合检索，适合知识关联 |
| **嵌入模型** | text-embedding-3-small | 成本低，性能足够，支持多语言 |
| **LLM** | hiclaw-gateway/qwen3.5-plus | 现有网关已集成，无需额外配置 |
| **存储** | MinIO | 现有基础设施，支持对象存储 |
| **前端框架** | React 18 + TypeScript | 生态成熟，类型安全，团队熟悉 |
| **构建工具** | Vite | 开发体验好，构建速度快 |
| **部署** | Docker + 本地服务器 | 可控性强，成本低 |
