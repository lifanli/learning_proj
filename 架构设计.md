# AI 知识库自动出版系统 - 架构设计

## 1. 项目愿景
构建一个**"自学型"AI 知识出版团队**。它不仅是机械的爬虫，更像是一个勤奋的学生兼专业的编辑团队。
*   **学生端 (Input)**：给定官方文档入口或公众号链接，它能自主顺藤摸瓜，阅读子链接，遇到 GitHub 项目会自动跳转阅读源码或 README，遇到死链会自动搜索替代。
*   **出版端 (Output)**：将学到的碎片化知识，经过 LightRAG 的深度关联和防幻觉审核，最终整理成一本逻辑严密、排版精美的本地 Markdown 知识全书。

## 2. 核心设计理念：Agentic Workflow (智能体工作流)

我们将系统设计为多智能体协作系统 (Multi-Agent System)，采用 **ReAct (Reasoning and Acting)** 模式。

### 角色分工
1.  **研究员 (Researcher/Student)**: 负责"自学"。给定一个入口（如 PyTorch 文档首页），它会像人一样浏览、点击感兴趣的链接、搜索不懂的概念。
2.  **翻译员 (Translator)**: 负责语言转换。自动检测非中文内容，翻译为专业中文，保留代码/URL/术语。
3.  **目录规划师 (StructurePlanner)**: 负责动态分类。基于 LLM 判断内容归属，可自动创建新分类。
4.  **档案员 (Archivist)**: 负责"记忆"。将研究员发回的笔记存入 LightRAG，构建知识图谱。
5.  **总编辑 (Chief Editor)**: 负责"统筹"。编排完整流水线，生成最终 Markdown 文档。
6.  **事实核查员 (FactChecker)**: 负责"核实"。验证 ArXiv ID、RAG 交叉验证、来源归属检查。

## 3. 模块化架构详解

### 3.1. 智能研究层 (The Researchers) - `src/researchers/`

*   **DocResearcher (文档自学者)**
    *   *输入*: 官方文档入口 URL
    *   *能力*:
        *   **递归阅读**: 识别页面中的子链接，BFS 遍历
        *   **增量学习**: 记录内容 SHA256 哈希，跳过未变化的页面（哈希存储于 `data/doc_hashes.json`）
    *   *工具*: `WebBrowser`

*   **WechatReader (公众号解析器)**
    *   *输入*: 微信公众号文章链接
    *   *能力*:
        *   **纯净提取**: 提取正文，去除广告
        *   **多模态图片解读**: 使用 Qwen-VL 解析技术图片
        *   **GitHub 链接嗅探与深度阅读**: 自动抓取 README 和代码结构
        *   **死链自愈**: 通过 WebSearch 工具搜索替代链接
    *   *工具*: `WebBrowser`, `GitHubAPI`, `WebSearch`

*   **ArxivResearcher (论文研究员)**
    *   *输入*: 关键词搜索 或 每日自动触发
    *   *能力*: 搜索论文 → 下载 PDF → 解析文本

### 3.2. 知识存储与索引层 (The Archivist) - `src/storage/`

核心：**LightRAG + Qwen-Plus**
*   **Graph Builder**: 利用 LLM 从文本中提取精细的实体关系
*   **Vector Store**: 辅助语义检索
*   **查询模式**: naive, local, global, hybrid

### 3.3. 智能体层 (The Agents) - `src/agents/`

*   **BaseAgent**: 所有智能体的基类，封装 LLM 调用、流式思考输出
*   **EditorAgent (总编辑)**: 编排完整处理流水线
*   **TranslatorAgent (翻译员)**: 自动检测语言并翻译为中文

### 3.4. 内容生成与出版层 (The Publisher) - `src/publisher/`

*   **StructurePlanner (目录规划器)**:
    *   动态维护 `config/toc.yaml`
    *   LLM 驱动的分类决策
    *   自动创建新分类目录
*   **FactChecker (事实核查)**:
    *   ArXiv ID 真实性验证（通过 ArXiv API）
    *   RAG 交叉验证（检测矛盾）
    *   来源归属检查
    *   自动修正问题内容

### 3.5. 工具层 (The Tools) - `src/tools/`

*   **WebSearch**: DuckDuckGo 搜索封装，支持通用搜索和首链获取
*   **WebBrowser**: 通用网页抓取（BeautifulSoup）
*   **GitHubAPI**: GitHub 仓库信息获取（PyGithub）
*   **PDFParser**: PDF 文本提取（PyMuPDF）

## 4. 完整处理流水线

```
输入 (URL/查询)
    │
    ▼
研究员 (Researcher) ── 抓取内容 + GitHub/图片解读
    │
    ▼
总编辑 (EditorAgent) 流水线:
    ├── 1. RAG 插入原始内容
    ├── 2. 翻译主题和摘要 (TranslatorAgent)
    ├── 3. 目录分类 (StructurePlanner)
    ├── 4. RAG 综合查询
    ├── 5. 翻译输出内容 (TranslatorAgent)
    ├── 6. LLM 编排最终文章
    ├── 7. 事实核查 (FactChecker)
    └── 8. 写入 Markdown 文件
    │
    ▼
知识库 (knowledge_base/) ── 分类目录 + Markdown 文章
```

## 5. 技术栈与配置

### 5.1. 大模型配置 (LLM)
*   **主模型**: `qwen3.5-plus` (通义千问) — 深度思考 + 文本生成
*   **多模态模型**: `qwen-vl-max` — 图片解读
*   **RAG 模型**: `qwen-plus` — 知识图谱构建
*   **Embedding**: `text-embedding-v2` (1536 维)
*   **Base URL**: `https://dashscope.aliyuncs.com/compatible-mode/v1`
*   **Features**: `enable_thinking: True` 开启深度思考模式

### 5.2. 核心依赖
*   **网络**: `requests`, `aiohttp`
*   **解析**: `beautifulsoup4`, `pymupdf`, `PyGithub`
*   **搜索**: `duckduckgo-search`
*   **RAG**: `lightrag`, `networkx`, `numpy`
*   **LLM**: `openai` (兼容接口)
*   **UI**: `streamlit`, `nest_asyncio`
*   **配置**: `pyyaml`, `python-dotenv`

## 6. Streamlit UI

四个功能页面：
1.  **知识库浏览**: 目录树导航 + Markdown 文章渲染
2.  **链接提交**: URL 输入 + 类型自动检测 + 加入处理队列
3.  **任务管理**: 待处理/已处理列表 + 一键处理按钮
4.  **运行日志**: 按日期查看日志 + 级别过滤 + 自动刷新

启动命令: `streamlit run app.py`

## 8. 使用方式

### CLI 模式
```bash
# 每日 ArXiv 更新
python main.py --mode daily

# 处理微信文章
python main.py --mode wechat --url <URL>

# 搜索 ArXiv 论文
python main.py --mode arxiv --query "DeepSeek"

# 爬取技术文档
python main.py --mode doc --url <URL>

# 处理队列中的所有链接
python main.py --mode queue
```

### Web UI 模式
```bash
streamlit run app.py
```
